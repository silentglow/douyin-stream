# Media Tools 前端重构设计文档

## 方案：iOS Widget 风格桌面

> **目标**：彻底抛弃现有前端界面，以 iOS 小组件（Widget）为核心设计语言，构建一个既像原生 Apple 应用、又高度实用的全新界面。
>
> **核心体验**：打开应用 → 一眼看到所有重要信息 → 点击小组件进入详情/操作。

---

## 一、信息架构：从 4 页到 3 页

现有界面有 4 个页面：发现、创作者、仪表盘、设置。信息分散，用户需要在多个页面间跳转才能掌握全局。

新架构只有 3 个顶级入口，符合 iOS 应用的设计模式：

| 页面 | 对应 iOS 应用 | 作用 |
|------|---------------|------|
| **工作台 (Home)** | 像 iOS 桌面 + 天气/股市小组件 | 打开应用看到的第一个画面。所有重要信息以小组件形式呈现，一目了然 |
| **内容库 (Library)** | 像 Apple Music 资料库 / Photos | 浏览所有已下载的视频和转写结果，支持搜索和筛选 |
| **设置 (Settings)** | 像 iOS 设置 | 账号配置、Cookie 管理、全局偏好 |

**为什么这样设计？**

- 用户 80% 的时间只需要"看看系统在干什么"和"启动新任务"，这两个需求在"工作台"一页内全部解决
- "发现"（粘贴链接下载）合并到工作台的一个快捷操作小组件中
- "创作者管理"合并到工作台的一个创作者状态小组件中
- "仪表盘"的所有数据（健康状态、任务统计、额度）全部拆分成独立的小组件，放在工作台

## 二、视觉设计系统

### 2.1 设计理念

**"信息即装饰"** —— 不需要额外的装饰元素，信息本身通过排版和色彩就构成美观的界面。

参考应用：
- **iOS 桌面小组件**：圆角矩形、背景模糊、信息层次
- **Apple Music**：大封面、沉浸式、简洁控制
- **iOS 天气**：根据内容自动变化的渐变背景

### 2.2 色彩系统

采用 Apple 原生系统色，不使用自定义品牌色：

```css
/* Light Mode */
--bg-primary: #F2F2F7;           /* 页面背景：iOS 系统灰 */
--bg-widget: #FFFFFF;             /* 小组件背景：纯白 */
--bg-widget-secondary: #F2F2F7;   /* 次要小组件背景 */
--text-primary: #000000;          /* 主文字 */
--text-secondary: #8E8E93;        /* 次要文字 */
--text-tertiary: #C7C7CC;         /* 第三级文字 */
--accent-blue: #007AFF;           /* 主要操作 */
--accent-green: #34C759;          /* 成功状态 */
--accent-orange: #FF9500;         /* 警告状态 */
--accent-red: #FF3B30;            /* 错误状态 */
--separator: rgba(0,0,0,0.08);    /* 分隔线 */

/* Dark Mode */
--bg-primary: #000000;
--bg-widget: #1C1C1E;
--bg-widget-secondary: #2C2C2E;
--text-primary: #FFFFFF;
--text-secondary: #8E8E93;
--accent-blue: #0A84FF;
--accent-green: #30D158;
--accent-orange: #FF9F0A;
--accent-red: #FF453A;
```

### 2.3 字体系统

使用系统默认字体栈（自动匹配 SF Pro / PingFang SC）：

```css
font-family: -apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text", "PingFang SC", "Helvetica Neue", sans-serif;
```

| 级别 | 大小 | 字重 | 用途 |
|------|------|------|------|
| Display | 34px | Bold (700) | 大型数据展示（如任务数量） |
| Title 1 | 28px | Semibold (600) | 页面标题 |
| Title 2 | 22px | Semibold (600) | 小组件标题 |
| Title 3 | 17px | Semibold (600) | 卡片标题、按钮 |
| Body | 15px | Regular (400) | 正文 |
| Callout | 14px | Regular (400) | 说明文字 |
| Caption | 12px | Regular (400) | 时间戳、辅助信息 |
| Footnote | 11px | Medium (500) | 标签、Badge |

### 2.4 间距系统

采用 iOS 的 8pt 网格系统：

- 小组件间隙：16px（桌面）/ 12px（手机）
- 小组件内边距：20px
- 文字间距：4px（紧凑）/ 8px（常规）/ 16px（宽松）
- 页面边距：24px（桌面）/ 16px（手机）

### 2.5 圆角系统

```css
--radius-widget: 22px;        /* 小组件圆角：iOS 桌面小组件标准 */
--radius-button: 12px;        /* 按钮圆角 */
--radius-pill: 9999px;        /* 胶囊按钮 */
--radius-icon: 14px;          /* 图标背景 */
```

### 2.6 阴影与层次

小组件不使用边框，使用极淡的阴影营造浮起感：

```css
/* Light Mode */
box-shadow: 0 2px 12px rgba(0, 0, 0, 0.06), 0 0 1px rgba(0, 0, 0, 0.04);

/* Dark Mode */
box-shadow: 0 2px 12px rgba(0, 0, 0, 0.3), 0 0 1px rgba(255, 255, 255, 0.04);
```

### 2.7 玻璃效果（Glassmorphism）

特定小组件（如系统状态、天气风格）使用背景模糊：

```css
background: rgba(255, 255, 255, 0.7);
backdrop-filter: blur(20px) saturate(180%);
-webkit-backdrop-filter: blur(20px) saturate(180%);
```

## 三、小组件系统（Widget System）

### 3.1 小组件尺寸

仿照 iOS 小组件的三种标准尺寸：

| 尺寸 | 桌面端宽度 | 移动端宽度 | 用途 |
|------|-----------|-----------|------|
| **Small** | 1/4 网格 | 1/2 网格 | 单数据展示（如"运行中任务数"） |
| **Medium** | 1/2 网格 | 满宽 | 列表或双数据（如"创作者状态"） |
| **Large** | 满宽 | 满宽 | 复杂信息（如"实时任务进度"） |

### 3.2 小组件设计规范

每个小组件包含：
1. **图标**（左上角，24x24，对应系统图标）
2. **标题**（图标右侧，Title 3 字号）
3. **主体内容**（中部，根据类型不同）
4. **底部状态**（可选，Caption 字号）

**禁止元素**：
- 按钮（小组件本身是可点击的，点击进入详情页）
- 分割线（用留白代替）
- 过多的颜色（最多使用一种强调色）

### 3.3 小组件清单

#### Small Widgets

**1. 任务计数 (Task Count)**
- 显示：运行中任务数量（大号数字 Display）
- 底部："运行中"文字
- 背景：根据数量变色（0=灰色，1-3=蓝色，3+=橙色）
- 点击：展开任务中心弹窗

**2. 健康状态 (Health Status)**
- 显示：彩色圆点 + "系统正常" / "X 个异常"
- 背景：绿色（正常）/ 红色（异常）的极淡 tint
- 点击：跳转健康详情

**3. Qwen 额度 (Qwen Quota)**
- 显示：剩余额度小时数（大号数字）
- 底部："可用额度" + 账号数量
- 背景：蓝色 tint
- 点击：跳转设置页账号池

**4. 创作者数量 (Creator Count)**
- 显示：关注的创作者数量
- 底部："X 个创作者 · Y 个自动同步"
- 点击：跳转内容库并筛选创作者

#### Medium Widgets

**5. 活跃任务列表 (Active Tasks)**
- 显示：最多 3 个正在运行的任务
- 每个任务：名称 + 进度条 + 百分比
- 底部："还有 X 个任务"（如果多于 3 个）
- 点击：展开完整任务中心

**6. 创作者快捷操作 (Creator Quick Actions)**
- 显示：最近同步过的 3 个创作者头像
- 底部："增量同步" / "全量同步" 快捷按钮（小组件内唯一允许按钮的情况）
- 点击：跳转内容库

**7. 最近动态 (Recent Activity)**
- 显示：最近 3 条动态（下载完成 / 转写失败 / 同步完成）
- 每条：图标 + 一句话描述 + 时间
- 点击：展开完整动态列表

**8. 失败摘要 (Failure Summary)**
- 显示：最近 24 小时的失败次数 + 主要原因
- 底部："查看详情"链接
- 背景：如果有失败，淡红色 tint

#### Large Widgets

**9. 实时任务进度 (Live Task Progress)**
- 显示：当前最活跃的一个任务的完整进度
- 包含：任务名称、阶段指示器（下载→转写→导出）、总体进度条、预计剩余时间
- 背景：渐变（下载阶段蓝色→转写阶段紫色→导出阶段绿色）
- 点击：展开任务详情面板

**10. 创作者概览 (Creator Overview)**
- 显示：创作者列表的精简版
- 每行：头像 + 名字 + 最后同步时间 + 资产数量
- 底部："管理创作者"按钮
- 点击：跳转内容库

**11. 快捷操作面板 (Quick Actions)**
- 显示：一排快捷操作按钮
  - 粘贴链接下载
  - 添加创作者
  - 开始本地转写
  - 清理历史任务
- 点击：直接触发操作或打开对应弹窗

## 四、页面设计

### 4.1 工作台 (Home) — 默认页

**桌面端布局**：
```
+----------------------------------------------------------+
|  Media Tools                           [搜索] [通知]      |
+----------------------------------------------------------+
|                                                          |
|  +------------------+  +------------------+             |
|  | 任务计数         |  | 健康状态         |             |
|  | 3                |  | ● 正常           |             |
|  +------------------+  +------------------+             |
|                                                          |
|  +----------------------------------------+             |
|  | 实时任务进度                            |             |
|  | 创作者同步 · ████████░░ 80%             |             |
|  +----------------------------------------+             |
|                                                          |
|  +------------------+  +------------------+             |
|  | Qwen 额度        |  | 创作者数量       |             |
|  | 12h              |  | 8                |             |
|  +------------------+  +------------------+             |
|                                                          |
|  +----------------------------------------+             |
|  | 快捷操作面板                            |             |
|  | [粘贴链接] [添加创作者] [本地转写] [清理] |             |
|  +----------------------------------------+             |
|                                                          |
|  +------------------+  +------------------+             |
|  | 最近动态         |  | 活跃任务列表     |             |
|  | ✓ 下载完成 x3    |  | 任务1 ████░░ 60% |             |
|  | ⚠ 转写失败 x1    |  | 任务2 ██░░░░ 30% |             |
|  +------------------+  +------------------+             |
|                                                          |
+----------------------------------------------------------+
```

**布局规则**：
- 桌面端：4 列网格，小组件自动排列
- 移动端：2 列网格（Small widget）或满宽（Medium/Large）
- 小组件之间间隙：16px
- 页面顶部没有标题栏（信息已经体现在小组件中）

**空状态**：
- 没有任务时，显示一个精美的空状态小组件：
  - 插图：一个睡着的云朵图标（SF Symbols: `cloud`）
  - 文字："系统空闲中"
  - 按钮："添加第一个创作者"

### 4.2 内容库 (Library)

**设计思路**：像 Apple Music 的资料库。

**顶部**：
- 分段控制器（Segmented Control）：全部 / 视频 / 转写文本
- 搜索栏（iOS 风格，圆角、灰色背景、放大镜图标）

**内容区域**：
- 创作者作为"专辑封面"展示：圆角矩形封面 + 创作者名字 + 视频数量
- 网格布局：桌面 4 列，平板 3 列，手机 2 列
- 每个卡片：封面图（或首字母头像）+ 名字 + 最新同步时间 + 资产数

**点击创作者**：
- 进入创作者详情页（像 Apple Music 的专辑页）
- 顶部大封面 + 创作者信息
- 下方是视频列表（像歌曲列表）
- 每个视频：缩略图 + 标题 + 状态（已转写 / 待转写 / 失败）

### 4.3 设置 (Settings)

**设计思路**：像 iOS 设置应用。

**布局**：
- 桌面端：左侧设置列表，右侧详情（像 macOS 设置）
- 移动端：全屏列表，点击进入子页面

**设置分组**：
```
账号配置
  ├── 抖音 Cookie
  ├── B 站 Cookie
  └── Qwen 账号池

全局偏好
  ├── 自动转写
  ├── 转写后删除视频
  ├── 导出格式 (MD / SRT)
  └── 并发数

系统
  ├── 清理历史任务
  ├── 日志目录
  └── 关于
```

**视觉风格**：
- 每个设置项：左侧图标 + 文字，右侧当前值或开关
- 分组使用大标题 + 留白分隔（不用分割线）
- 开关使用 iOS 风格（圆角药丸开关）

## 五、导航系统

### 5.1 桌面端

- **左侧边栏**：固定宽度 200px，毛玻璃效果
  - 图标 + 文字导航项：工作台、内容库、设置
  - 当前选中的项：蓝色背景 pill + 蓝色图标
  - 底部：主题切换（浅色/深色）

### 5.2 移动端

- **底部 Tab 栏**：固定高度 64px + safe area
  - 三个 Tab：工作台、内容库、设置
  - 图标使用 SF Symbols 风格（线条感）
  - 选中状态：图标填充 + 蓝色

## 六、动画与交互

### 6.1 页面切换

- 类型：共享元素过渡（Shared Element Transition）
- 效果：点击小组件 → 小组件平滑放大填满屏幕 → 详情页内容淡入
- 时长：300ms
- 缓动：cubic-bezier(0.32, 0.72, 0, 1)（iOS 风格弹簧）

### 6.2 小组件出现

- 初始状态：opacity 0, translateY(20px), scale(0.95)
- 结束状态：opacity 1, translateY(0), scale(1)
- 时长：400ms
-  stagger：每个小组件间隔 50ms 依次出现

### 6.3 数据更新

- 数字变化：使用动画计数（从旧值滚动到新值）
- 进度条：平滑过渡，width 变化使用 500ms ease-out
- 状态变化：颜色渐变过渡，300ms

### 6.4 点击反馈

- 小组件按下：scale(0.97)，200ms
- 按钮按下：scale(0.95) + 背景变深，100ms
- 列表项滑动：iOS 风格左滑删除（手机端）

### 6.5 骨架屏

- 加载时：小组件显示为灰色圆角矩形
- Shimmer 效果：从左到右的白色渐变扫过
- 时长：1.5s 循环

## 七、响应式断点

| 断点 | 设备 | 网格列数 | 小组件尺寸 |
|------|------|---------|-----------|
| < 640px | 手机 | 2 列 | Small=1x1, Medium/Large=满宽 |
| 640-1024px | 平板 | 3 列 | Small=1x1, Medium=1x2, Large=满宽 |
| > 1024px | 桌面 | 4 列 | Small=1x1, Medium=1x2, Large=1x4 |

## 八、暗黑模式

- 跟随系统设置（prefers-color-scheme）
- 小组件背景从纯白变为 #1C1C1E
- 文字从黑色变为白色
- 强调色自动切换（蓝色 #007AFF → #0A84FF）
- 阴影在暗黑模式下更深

## 九、数据结构

小组件需要从后端获取的数据：

```typescript
// 工作台数据
interface HomeData {
  activeTasks: Task[];           // 活跃任务（用于任务计数、任务列表、进度）
  healthStatus: HealthCheck;     // 健康状态
  qwenQuota: QuotaStatus;        // Qwen 额度
  creators: Creator[];           // 创作者列表
  recentActivity: Activity[];    // 最近动态
  failureSummary: FailureSummary; // 失败摘要
}

// 每个小组件独立刷新，互不影响
```

1. **搭建基础框架**：新的路由结构、布局组件、暗黑模式支持
